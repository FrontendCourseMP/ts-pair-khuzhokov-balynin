# Библиотеки для валидации:

### 1\. Zod

- **Тип**: Schema-first
- **Интеграция с формами**: 5/5
- **Удобство схемы**: 5/5
- **Примечание**: Лучшая поддержка TS и вывод типов.

### 2\. Yup

- **Тип**: Schema-first

- **Интеграция с формами**: 5/5

- **Удобство схемы**: 5/5

- **Примечание**: Одна из самых зрелых и популярных библиотек. Использует цепной синтаксис на основе объектов, похожий на Joi. Является стандартом при работе с Formik и хорошо интегрируется с React Hook Form. Отлично работает с TypeScript, хотя уступает Zod по глубине вывода типов.

### 3\. Joi

- **Тип**: Schema-first

- **Интеграция с формами**: 5/5

- **Удобство схемы**: 5/5

- **Примечание**: Изначально разработан и широко используется в Node.js для валидации API. Может использоваться на фронтенде, но его главный акцент — на серверной валидации. Имеет очень богатый набор правил и гибкую систему ошибок.

### 4\. JustValidate

- **Тип**: Field-first (Vanilla JS)

- **Интеграция с формами**: n/a (не требует)

- **Удобство схемы**: n/a (использует императивный подход .addField())

- **Примечание**: Легковесная библиотека без зависимостей (Vanilla JavaScript). Идеально подходит для проектов, которые не используют большие фреймворки (React, Vue, Angular) или для простых статических форм, где требуется минимальный размер бандла. Валидация привязывается напрямую к DOM-элементам, а не к схеме данных.

### 5\. Valibot

- **Тип**: Schema-first

- **Интеграция с формами**: n/a (быстро растет)

- **Удобство схемы**: 5/5

- **Примечание**: Новый и очень быстрый конкурент Zod. Обеспечивает отличную поддержку TypeScript и является одной из самых легких библиотек по размеру бандла. Использует более функциональный подход к определению схемы, чем объектный подход Zod/Yup.

# JustValidate: Точка входа и начальные команды

## Начальные команды, точка входа и базовая инициализация

### Установка через npm:

```bash
npm install just-validate
```

Точкой входа является создание нового экземпляра `Just Validate`, передавая ему селектор или
Dom-элемент вашей формы

### Пример:

```HTML
<form id="my_form">
  <input type="text" id="name" name="name" placeholder="Имя" />
  <input type="email" id="email" name="email" placeholder="Email" />
  <button type="submit">Отправить</button>
</form>
```

```JavaScript
import JustValidate from 'just-validate';

const validator = new JustValidate('#my_form');

validator
  .addField('#name', [
    {
      rule: 'required',
      errorMessage: 'Поле "Имя" обязательно',
    },
    {
      rule: 'minLength',
      value: 3,
      errorMessage: 'Минимум 3 символа',
    },
  ])

  .addField('#email', [
    {
      rule: 'required',
      errorMessage: 'Поле "Email" обязательно',
    },
    {
      rule: 'email',
      errorMessage: 'Введите корректный Email',
    },
  ])
  .onSuccess((event) => {
    console.log('Валидация прошла успешно. Данные формы:', event.target);
  });
```

## Основные методы

- `new JustValidate(form, globalConfig, distLocale)`:
  - `form`: Селектор или Dom-элемент формы.
  - `globalConfig`: Необязательный объект глобальных настроек (например, настройки для отображения ошибок).
  - `distLocale`: Необязательный объект для пользовательской локализации.
- `.addField(selector, rules, config)`: Метод для добавления поля и правил валидации к нему
- `.addRequiredGroup(selector, errorMessage)`: Метод для проверки того, что хотя бы один элемент
  в группе (чекбоксы, радио) выбран.
- `.onSuccess(callback)`: Метод, который регистрирует функцию-колбэк, выполняемую только в случае успешной
  валидации всей формы при попытке отправки.
